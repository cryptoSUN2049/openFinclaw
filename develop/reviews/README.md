# Code Review 记录索引 & 规范

## 1. Code Review 规范

- 所有合并到 dev/main 的代码必须经过 Review
- 至少 1 位 Reviewer approve
- Review 重点：功能正确性、安全性、性能、可维护性、代码风格

## 2. 严重级别定义

| 级别          | 说明                   | 处理方式           |
| ------------- | ---------------------- | ------------------ |
| 🔴 Critical   | 安全漏洞、数据丢失风险 | 必须修复后才能合并 |
| 🟠 Major      | 功能缺陷、性能问题     | 强烈建议修复       |
| 🟡 Minor      | 代码风格、命名不规范   | 建议修复           |
| 🔵 Suggestion | 改进建议、优化方向     | 可选               |

## 3. Review 检查清单

- [ ] 功能是否按需求实现
- [ ] 是否有潜在安全问题（XSS、SQL 注入、CSRF 等）
- [ ] 错误处理是否完善
- [ ] 是否有性能隐患
- [ ] 代码是否可读可维护
- [ ] 是否有测试覆盖
- [ ] TypeScript 类型是否正确
- [ ] 是否遵循项目编码规范

## 4. Review 报告格式

- **文件命名**：`v{版本}-{模块}-review.md`
- **存放路径**：`reviews/{日期}/`
- 必须包含 YAML frontmatter
