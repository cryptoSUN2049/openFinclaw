# Arena 竞技引擎验收标准

## 功能验收

- [x] 排行榜展示：按 ELO 分数排序，展示段位、胜率、收益率等指标
- [x] ELO 计算正确性：对决结果正确更新双方 ELO 分数，K 值随段位递减
- [x] 策略 PK 对决：选择对手 → 提交策略 → 回测执行 → 结果判定
- [x] 段位分配准确：根据 ELO 分数正确分配青铜至王者段位
- [x] SSE 事件推送：对决过程实时推送状态变更（排队、执行中、完成）
- [x] 提交状态机：策略提交经过 pending → scanning → backtesting → completed 完整流转
- [x] 安全扫描：策略代码提交后自动执行规则 + LLM 安全检测
- [ ] Arena 提交页面：用户可通过 UI 提交策略代码
- [ ] 对战详情页：展示对决双方策略表现、收益曲线对比
- [ ] ELO 历史图表：展示用户 ELO 分数变化趋势
- [ ] 实时 PK 观战：其他用户可观看正在进行的对决
- [ ] 策略代码编辑器集成：内置代码编辑器支持策略编写

## 安全验收

- [x] 策略代码沙箱执行
- [x] 恶意代码检测（规则 + LLM）
- [ ] 提交频率限制
- [ ] 策略代码大小限制校验

## UI 验收

- [x] 排行榜页面响应式布局
- [x] 段位徽章正确渲染
- [x] 排行榜 SSR 首屏加载性能
- [ ] 提交页面表单校验与错误提示
- [ ] 对战详情页数据可视化

## 性能验收

- [x] 排行榜 SSR 首屏加载 < 2s
- [ ] SSE 连接稳定性（断线重连）
- [ ] 回测任务队列不阻塞主流程
- [ ] 并发对决场景下 ELO 计算一致性

## 测试验收

- [ ] ELO 引擎单元测试覆盖 ≥90%
- [ ] 提交状态机单元测试
- [ ] 安全扫描器单元测试
- [ ] E2E 策略提交流程测试
- [ ] E2E 排行榜展示测试
